# 功能需求文档

## 项目概述

行业/产品机会探测器是一个多信号融合系统，通过整合多个免费数据源来识别行业机会和产品机会。

## 核心功能需求

### 1. 数据采集功能

#### 1.1 新闻数据采集
- **需求编号**: FUNC-001
- **描述**: 从 GDELT Doc API 采集新闻数据
- **输入**: 主题关键词、时间范围
- **输出**: 新闻标题、链接、发布时间、热度指标
- **优先级**: 高
- **状态**: 已实现

#### 1.2 开发者讨论采集
- **需求编号**: FUNC-002
- **描述**: 从 Hacker News Algolia API 采集开发者讨论
- **输入**: 主题关键词、时间范围
- **输出**: 讨论标题、链接、讨论热度、评论数
- **优先级**: 高
- **状态**: 已实现

#### 1.3 开源生态数据采集
- **需求编号**: FUNC-003
- **描述**: 从 GitHub Search API 采集开源项目信息
- **输入**: 主题关键词、时间范围
- **输出**: 项目数量、star 数、最近活跃度
- **优先级**: 高
- **状态**: 已实现

#### 1.4 用户讨论采集
- **需求编号**: FUNC-004
- **描述**: 从 Reddit 公共 JSON API 采集用户讨论
- **输入**: 主题关键词、时间范围
- **输出**: 帖子标题、链接、讨论热度、评论数
- **优先级**: 中
- **状态**: 已实现

#### 1.5 研究论文采集
- **需求编号**: FUNC-005
- **描述**: 从 arXiv API 采集相关研究论文（基础元数据）
- **输入**: 主题关键词、时间范围
- **输出**: 论文标题、摘要（精简）、作者、发布时间、链接
- **优先级**: 中
- **状态**: 部分实现（当前未下载论文全文）

#### 1.6 研究论文下载与总结（增强）
- **需求编号**: FUNC-005A
- **描述**: 支持下载论文 PDF（优先 arXiv PDF），并做本地缓存，供后续提取/阅读
- **输入**: arXiv 论文链接（或 arXiv id）、下载目录、缓存策略
- **输出**: PDF 文件路径、下载状态、基础元信息（页数/大小）
- **优先级**: 中
- **状态**: 待实现

- **需求编号**: FUNC-005B
- **描述**: 对论文进行总结（优先基于全文/前几页 + 摘要），输出“可落地”摘要
- **输入**: 论文 PDF/文本、摘要、主题、摘要粒度（TL;DR/要点/可用结论/潜在产品化方向）
- **输出**: 结构化总结（贡献点、方法、结论、可产品化点、风险/局限）
- **优先级**: 中
- **状态**: 待实现（可接入本地/远程大模型，如 Ollama）

### 2. 数据处理功能

#### 2.1 信号计算
- **需求编号**: FUNC-006
- **描述**: 计算需求、动量、竞争三个核心信号
- **输入**: 采集的原始数据
- **输出**: 标准化后的信号值
- **优先级**: 高
- **状态**: 已实现

#### 2.2 机会分数计算
- **需求编号**: FUNC-007
- **描述**: 基于加权公式计算机会分数
- **输入**: 三个核心信号值、权重配置
- **输出**: 机会分数（0-1 范围）
- **优先级**: 高
- **状态**: 已实现

#### 2.3 数据标准化
- **需求编号**: FUNC-008
- **描述**: 对不同数据源的数据进行标准化处理
- **输入**: 原始数据
- **输出**: 标准化后的数据格式
- **优先级**: 高
- **状态**: 已实现

### 3. 洞察生成功能

#### 3.1 行业洞察生成
- **需求编号**: FUNC-009
- **描述**: 基于数据生成结构化行业洞察
- **输入**: 处理后的信号数据
- **输出**: 洞察类型、证据、建议动作、置信度
- **优先级**: 高
- **状态**: 已实现

#### 3.2 机会类型识别
- **需求编号**: FUNC-010
- **描述**: 识别不同类型的机会（新兴、增长、成熟等）
- **输入**: 信号数据和趋势分析
- **输出**: 机会类型分类
- **优先级**: 中
- **状态**: 已实现

### 4. 报告生成功能

#### 4.1 CSV 报告生成
- **需求编号**: FUNC-011
- **描述**: 生成结构化的 CSV 格式报告
- **输入**: 信号数据、机会分数、洞察数据
- **输出**: signals.csv、opportunities.csv、insights.csv、events.csv
- **优先级**: 高
- **状态**: 已实现

#### 4.2 Markdown 报告生成
- **需求编号**: FUNC-012
- **描述**: 生成可读的 Markdown 格式报告
- **输入**: 信号数据、机会分数、洞察数据
- **输出**: report.md、insights.md、daily.md
- **优先级**: 高
- **状态**: 已实现

#### 4.3 每日报告生成
- **需求编号**: FUNC-013
- **描述**: 生成每日风格的简报和归档（强调“机会洞察在前，事件证据在后”）
- **输入**: 当日数据、历史对比
- **输出**: daily/YYYY-MM-DD.md、daily_report.md
- **优先级**: 中
- **状态**: 已实现

#### 4.4 每日报告结构与栏目（增强）
- **需求编号**: FUNC-013A
- **描述**: 调整 `每日行业事件与机会洞察` 报告结构，把“机会洞察/可执行建议”放在最前面，行业事件作为证据附录放后面
- **输入**: 洞察、事件列表、栏目定义
- **输出**: 结构更聚焦的 daily_report.md
- **优先级**: 中
- **状态**: 已实现

- **需求编号**: FUNC-013B
- **描述**: 行业事件按“大模块/栏目”分组输出（产品发布、融资并购、政策监管、开源发布、研究论文、价格与商业模式、安全事件等）
- **输入**: 事件列表、分类规则
- **输出**: 按栏目分组的事件表格（每栏 Top + 原因）
- **优先级**: 中
- **状态**: 已实现（但依赖数据源覆盖度）

## 功能优先级矩阵

| 功能类别 | 高优先级 | 中优先级 | 低优先级 |
|---------|---------|---------|---------|
| 数据采集 | GDELT, HN, GitHub | Reddit, arXiv | - |
| 数据处理 | 信号计算, 机会分数 | - | 高级算法 |
| 洞察生成 | 基本洞察 | 机会类型识别 | 高级分析 |
| 报告生成 | CSV, Markdown | 每日报告 | 可视化 |
| 配置管理 | 主题, 时间, 权重 | - | 高级配置 |
| 界面交互 | CLI | - | Web, API |

## 验收标准

1. 所有高优先级功能必须完全实现并通过测试
2. 中优先级功能应基本实现，允许少量缺陷
3. 低优先级功能可在后续迭代中实现
4. 所有功能必须提供适当的错误处理和日志记录
5. 生成的报告必须准确、可读且格式正确
