# 问题状态说明

## 状态流转

```
待处理 → 处理中 → 已解决 → 已关闭
```

## 状态详细说明

### 待处理 (Pending)

问题刚提交，尚未被调查或处理。通常由以下情况触发：

- 用户首次报告问题
- 问题需要进一步确认是否为有效问题
- 问题需要分配给相关人员

### 处理中 (In Progress)

问题正在被调查和解决。通常包括：

- 分析问题原因
- 复现问题
- 设计解决方案
- 实施修复

### 已解决 (Resolved)

问题已修复，等待验证。通常包括：

- 修复代码已提交
- 相关文档已更新
- 等待用户或测试验证

### 已关闭 (Closed)

问题已解决并验证通过。通常包括：

- 用户确认问题已解决
- 测试用例通过
- 问题记录归档

## 状态更新规则

1. **从待处理到处理中**: 当问题被确认为有效问题，并开始调查时
2. **从处理中到已解决**: 当修复代码已提交并通过初步测试时
3. **从已解决到已关闭**: 当用户或测试验证修复有效时
4. **从处理中到待处理**: 当问题需要更多信息或重新评估时
5. **从已解决到处理中**: 当验证发现修复不完整时

## 状态标记

在问题文件中使用以下格式标记状态：

```markdown
- **问题状态**: [x] 已解决
```

或

```markdown
- **问题状态**: 
  - [ ] 待处理
  - [ ] 处理中
  - [x] 已解决
  - [ ] 已关闭
```
