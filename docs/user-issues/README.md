# 用户问题记录

本文件夹用于记录用户提出的问题，包括问题描述、复现步骤、解决方案和状态跟踪。

## 文件结构

```
user-issues/
├── README.md              # 本文件
├── templates/             # 问题记录模板
│   ├── issue-template.md  # 问题记录模板
│   └── status-guide.md    # 状态说明
├── issues/                # 问题记录文件
│   ├── ISSUE-001.md       # 问题编号从 ISSUE-001 开始
│   └── ...
└── index.md               # 问题索引
```

## 问题记录格式

每个问题应该包含以下信息：

1. **问题标题** - 简明扼要地描述问题
2. **问题描述** - 详细说明问题是什么
3. **复现步骤** - 如何复现该问题
4. **预期结果** - 正常情况下应该发生什么
5. **实际结果** - 实际发生了什么
6. **环境信息** - 操作系统、Python版本、相关配置等
7. **解决方案** - 如何解决该问题
8. **状态** - 问题当前状态（待处理/处理中/已解决/已关闭）

## 状态说明

- `待处理` - 问题刚提交，尚未处理
- `处理中` - 问题正在被调查和解决
- `已解决` - 问题已修复，等待验证
- `已关闭` - 问题已解决并验证通过

## 使用方法

1. 当用户提出问题时，在 `issues/` 目录下创建一个新的问题文件
2. 使用 [`issue-template.md`](templates/issue-template.md) 作为模板填写问题详情
3. 在 [`index.md`](index.md) 中添加问题索引
4. 更新问题状态时，更新对应的问题文件
